# 安全策略

[English](SECURITY.md) | 简体中文

## 支持的版本

我们为以下版本发布安全漏洞补丁：

| 版本 | 支持状态 |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## 报告漏洞

如果您在本项目中发现安全漏洞，请发送电子邮件给维护者。所有安全漏洞都将得到及时处理。

**请不要通过公开的 GitHub Issues 报告安全漏洞。**

### 报告内容

报告漏洞时，请包含：

1. 漏洞描述
2. 重现问题的步骤
3. 漏洞的潜在影响
4. 建议的修复方案（如果有）

### 响应时间表

- **初始响应**：48 小时内
- **状态更新**：7 天内
- **修复时间表**：取决于严重程度
  - 严重：7 天内
  - 高：14 天内
  - 中：30 天内
  - 低：下一个常规版本

## 安全最佳实践

使用本库时：

1. 始终使用最新的稳定版本
2. 保持依赖项更新
3. 对数据库访问遵循最小权限原则
4. 在空间查询中使用用户输入之前验证和清理所有输入
5. 使用参数化查询防止 SQL 注入
6. 定期查看 [OWASP Top 10](https://owasp.org/www-project-top-ten/)

## 已知安全注意事项

### SQL 注入

本库使用 MyBatis Plus TypeHandlers，它们正确处理参数绑定。但是：

- 始终使用参数化查询
- 永远不要将用户输入直接拼接到 SQL 字符串中
- 谨慎使用动态 SQL 生成

### 数据验证

- 在创建几何对象之前验证坐标范围
- 检查坐标中的 NaN 和无限值
- 验证 SRID 值与数据库配置匹配

## 披露政策

当报告并确认安全漏洞时：

1. 我们将私下修复
2. 我们将准备安全公告
3. 我们将发布修补版本
4. 我们将发布安全公告，并向报告者致谢（如果需要）

感谢您帮助保持本项目的安全！
